中西教育--我的胡诌

发表于 2017-01-30

写machine translation写到一半突然迸发出一些奇怪的想法，记录下来，聊以慰己。

国内四年本科计算机，国外研究生一年的计算机，我的直接感受是：国内的高等教育（至少计算机这一块儿）相差国外，以十年计。在国内，大一上之后，便很少见一门课超过百分之五十以上的人全神贯注的听课，玩手机者有之，看剧者有之，抄作业者有之。鄙人不才，忝列其中。平心而论，我所在大学不算太好，但也不算太差，还是选取了一些高考中的佼佼者。我自己呢，从小到大也算好学，尤其到大三大四一年，深深愧疚于之前的荒嬉，可算是拼命补了一些课程。反观国外，学习上三军用命，每节lecture座无虚席，老师每问一个问题，几乎还没碰着冷场的情况，整个课堂的参与感与实际效果简直比国内好了太多。

之前写过一篇文章，很简单的写过自己在英国的这一年的教育体验，寥寥数语，现在看来简直不成文章，所以还是想要另写一篇，将感受都能够记录在这个里面，又因为思想并不受一时一地的限制，于是准备将这篇文章写成日记的形式，随时更新。

30/01/2017-计算机教育

一个词用以概括最强烈的反差—-“应用至上”，杜威的实用主义在西方破产了，但是他的基因却留下来了。单说一件小事：

知乎上的问题

“如何评价清华大学计算机系大二 java 小学期考试题？”

    public void test(Object o) {
        System.out.println("Object");
    }
    public void test(String s) {
        System.out.println("String");
    }
    public static void main(String[] args) {
        Main that = new Main();
        that.test(null);
    }
}

请写出运行输出
考试为闭卷书面考试.
如何看待这种要求初学者了解语言 edge case, 记忆 API 名称的命题方法?

英国的Java，无考试，两个大作业：

Assignments
The course assessment is based on two practical programming assignments (assignment1 and assignment2) which provide a realistic software development experience.
First assignment 30% First practical programming assignment
Second Assignment 70% Second practical programming assignment

除此之外，项目全程清晰而明了的讲解了一步一步的如何搭建一个实际生活中的工程项目，比起国内的教学模式，上手就能操作，趣味盎然。更为难得的是，国外并不以教学语言本身为目的，Java是很多课程的前置课程，列入了不同方向的学习路线中，而国内的话，个人感受，学了C语言，考试是最大的用途了（当然，C语言的学习方便了我理解操作系统，编译原理这些课程），但是大一进校就接触C语言，考试考语法细节，无任何工程实际的项目，天天刷算法题，课程没有路线规划，一股脑的给学生安上，在短短的两个月时间完全磨灭了我对CS本身的兴趣。

国外工程实践与理论结合的细节很多，以后再补充。
